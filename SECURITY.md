# Security Policy

StickyDock values responsible disclosure of security issues. If you discover a
security vulnerability, please report it so that it can be addressed safely and
quickly.

## Reporting a Vulnerability

- Open a GitHub issue with the title prefixed by `[SECURITY]`.
- Provide enough detail to reproduce the issue without sharing sensitive data.
- Do not post proof-of-concept exploits in public issue descriptions.

If GitHub Security Advisories are enabled for this repository, a private
advisory may be used instead.

## What to Include

- Affected version or branch
- Steps to reproduce
- Expected behavior
- Actual behavior
- Any relevant logs or error messages

## Supported Versions

Security fixes are accepted for the `main` branch and the latest release
series. If you are unsure whether a particular version is supported, please
ask in the security report.

## Disclaimer

Do not include personal, financial, or otherwise sensitive information in your
report. If we need additional data to reproduce a vulnerability, we will ask
for it securely.
